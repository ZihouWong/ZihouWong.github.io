# CascadingVC 里的 Pager 保留各个子 ViewController 的 contentOffset

## 简介

**现在项目里存在一个问题，在不同的 Page 所使用的是同一个 contentOffset。**

## 问题出现的原因

- 为了方便后续的描述，现在把 A 假设为开始滑动时的 Page，把 B 假设为滑动结束时的 Page。

- 前提：CascadingVC 在 `scrollViewDidScroll()` 中调用了 `layoutScrollView()` 函数，导致所有使用了 CascadingVC 的页面，在滑动过程中，会不断触发 `layoutScrollView()`

``` swift
extension CascadingVC: UIScrollViewDelegate {
    open func scrollViewDidScroll(_ scrollView: UIScrollView) {
        layoutScrollView(includingComponentsNotInVisualRange: false)
    }
}
```

当从 A 滑动到 B 时，由于 `layoutScrollView()` 函数 **不能** 保证其是一个原子操作，因此就造成了 当 A 切换到 B 的瞬间，A 触发了 `layoutScrollView()`函数，但是本该返回给 A 的结果，却返回给了 B。

## 解决的思路

切换页面有两种办法，第一种是拖动页面，第二种是点击上沿的按钮切换
